# 扫雷网页版 — 设计说明

## 1. 项目概述

本项目是一个纯前端的 Windows 经典扫雷复刻版，运行在浏览器中。游戏规则与 Windows 扫雷保持一致，UI 采用现代深色风格，不刻意模仿 Windows 像素界面。

| 项目 | 说明 |
|------|------|
| 技术栈 | HTML + CSS + JavaScript（Vanilla JS，无框架） |
| 运行环境 | 现代浏览器（Chrome、Edge、Firefox 等） |
| 部署方式 | 静态文件，无需后端 |
| 音效 | 无 |

---

## 2. 功能范围

### 2.1 已实现

- 三档预设难度（初级 / 中级 / 高级）
- 自定义难度（宽、高、雷数可调）
- 左键揭开、右键插/拔旗
- 数字格 Chord 双键展开（左键 + 右键同时按，或中键）
- 数字格双击展开（周围旗数等于数字时，揭开周围未插旗格）
- 首击保护（首格及周围 8 格无雷）
- 空白格 BFS 连锁展开
- 计时器（首击后开始，最长 999 秒）
- 雷数计数器（总雷数 − 插旗数，可为负）
- 胜负判定与状态栏表情反馈
- 新游戏 / 切换难度重置

### 2.2 明确不包含

- 音效
- 排行榜、存档、联网
- 问号标记（仅支持「无标记 ↔ 小旗」两态）

## 3. 难度参数

| 难度 | 行 × 列 | 雷数 |
|------|---------|------|
| 初级 | 9 × 9 | 10 |
| 中级 | 16 × 16 | 40 |
| 高级 | 16 × 30 | 99 |
| 自定义 | 用户输入 | 用户输入 |

**自定义约束：**

- 宽度：9 ~ 30
- 高度：9 ~ 24
- 雷数：1 ~ (宽 × 高 − 9)

非法参数会显示错误提示，不会开始新局。

## 4. 游戏规则

### 4.1 状态机

```
新游戏/选难度 → 等待首击 → 游戏中 → 胜利 / 失败 → 重开
```

| 状态 | 说明 |
|------|------|
| `idle` | 已开局，等待第一次左键点击 |
| `playing` | 游戏中 |
| `won` | 所有非雷格已揭开 |
| `lost` | 踩到雷，显示全部雷并高亮触发格 |

### 4.2 核心规则

1. **首击保护**：第一次左键点击的格子及其周围 8 格均不放雷；布雷在首击时进行，而非开局时。
2. **数字格**：揭开后的格子显示周围 8 格雷数（1~8）；为 0 时显示空白并自动扩散揭开相邻区域。
3. **插旗**：右键在未揭开格子上切换小旗；已揭开格子右键无效；已插旗格子左键无效。
4. **Chord / 双击**：在已揭开的数字格上，若周围小旗数等于该格数字，可通过以下方式自动揭开周围未插旗的隐藏格；若揭开的是雷则游戏失败。
   - 同时按下左键 + 右键（或中键）
   - **双击**该数字格
5. **胜负**：踩雷失败并显示所有雷；所有非雷格揭开则胜利。

## 5. 技术架构

### 5.1 模块划分

```
minesweeper/
├── index.html      # 页面结构
├── css/style.css   # 样式
└── js/
    ├── game.js     # 核心逻辑（与 DOM 无关）
    ├── ui.js       # 棋盘渲染、鼠标事件
    └── main.js     # 入口、难度切换、计时器
```

| 模块 | 职责 |
|------|------|
| `game.js` | 棋盘数据、布雷、揭开、插旗、Chord、胜负判定 |
| `ui.js` | CSS Grid 渲染、格子样式、鼠标/Chord 事件绑定 |
| `main.js` | 应用入口、难度切换、计时器、协调 game 与 ui |

逻辑层与 UI 层分离：`Game` 类不依赖 DOM，便于单元测试。

### 5.2 数据模型

**Cell（单格）：**

```javascript
{
  isMine: boolean,       // 是否为雷
  isRevealed: boolean,   // 是否已揭开
  isFlagged: boolean,    // 是否插旗
  neighborMines: number  // 周围雷数 0~8，布雷后预计算
}
```

**Game 状态：**

```javascript
{
  rows, cols, mineCount,
  board: Cell[][],
  status: 'idle' | 'playing' | 'won' | 'lost',
  flagsPlaced: number,
  firstClickDone: boolean,
  triggeredMine: [row, col] | null  // 失败时触发的雷格
}
```

### 5.3 关键算法

| 函数 | 说明 |
|------|------|
| `placeMines(safeRow, safeCol)` | 排除首击 3×3 区域后随机布雷，再计算 `neighborMines` |
| `revealCell(r, c)` | 揭开单格；若为 0 则用 BFS 队列扩散 |
| `toggleFlag(r, c)` | 切换插旗状态 |
| `chordReveal(r, c)` | 校验周围旗数 = 数字后批量 `revealCell` |
| `checkWin()` | 所有非雷格 `isRevealed` 则胜利 |

### 5.4 事件处理

- **右键**：`contextmenu` 事件 `preventDefault()` 阻止浏览器默认菜单。
- **Chord**：`mousedown` 中用位掩码检测左键（button 0）与右键（button 2）同时按下；中键（button 1）也可触发。
- **双击**：`dblclick` 事件复用 `chordReveal` 逻辑。
- **胜负后**：棋盘按钮 `disabled`，禁止继续操作。

## 6. UI 设计

### 6.1 布局

```
┌─────────────────────────────────────┐
│  标题 + 操作提示                      │
├─────────────────────────────────────┤
│  [初级][中级][高级][自定义]  [新游戏]  │
├─────────────────────────────────────┤
│  自定义难度表单（选「自定义」时显示）    │
├─────────────────────────────────────┤
│  🙂/💀    雷数: 010    计时: 000     │
├─────────────────────────────────────┤
│              棋盘 Grid               │
└─────────────────────────────────────┘
```

### 6.2 视觉风格

- **整体**：深色渐变背景（`#1a2332` ~ `#2d3a4f`），现代扁平风格。
- **未揭开格**：凸起按钮感（渐变 + 内阴影）。
- **已揭开格**：浅灰平面（`#c8d0da`）。
- **数字 1~8**：沿用经典扫雷配色（蓝、绿、红等）。
- **插旗**：🚩 emoji。
- **踩雷**：红色背景 + 💣；触发格额外高亮。
- **状态栏表情**：🙂 待机 → 😐 游戏中 → 😎 胜利 → 💀 失败。

### 6.3 响应式

格子大小随列数自动缩小：

| 列数 | 格子边长 |
|------|----------|
| ≤ 9 | 36px |
| ≤ 16 | 28px |
| ≤ 24 | 22px |
| > 24 | 18px |

高级（30 列）棋盘较宽时，外层容器支持横向滚动。

## 7. 测试

`tests/game.test.js` 提供核心逻辑自动化测试，覆盖：

- 首击安全区
- 插旗 / 取消插旗
- 已插旗格不可揭开
- 自定义难度校验
- 三档预设参数
- Chord 展开

运行方式（需 Node.js）：

```bash
cd minesweeper
node tests/game.test.js
```
