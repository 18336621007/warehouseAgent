# 扫雷网页版 — 使用说明

## 1. 启动游戏

本项目为纯静态网页，需通过 HTTP 访问（ES Module 在 `file://` 协议下可能无法加载）。

### 方式一：Python 内置服务器（推荐）

在项目目录下打开终端，执行：

```powershell
cd d:\code\Project\test\minesweeper
py -3 -m http.server 8080
```

浏览器访问：**http://localhost:8080**

### 方式二：其他静态服务器

也可使用 VS Code Live Server、Node.js `npx serve` 等任意静态文件服务，将根目录指向 `minesweeper` 文件夹即可。

## 3. 难度选择

| 难度 | 棋盘大小 | 雷数 |
|------|----------|------|
| 初级 | 9 × 9 | 10 |
| 中级 | 16 × 16 | 40 |
| 高级 | 16 × 30 | 99 |
| 自定义 | 见下表 | 见下表 |

**自定义难度范围：**

| 参数 | 范围 |
|------|------|
| 宽度 | 9 ~ 30 |
| 高度 | 9 ~ 24 |
| 雷数 | 1 ~ (宽 × 高 − 9) |

设置自定义难度后，点击 **「应用」** 或 **「新游戏」** 开始。参数不合法时会显示红色错误提示，不会开局。

切换难度会自动开始新一局（选「自定义」时仅显示设置面板，需手动应用或点新游戏）。

## 4. 鼠标操作

| 操作 | 作用 |
|------|------|
| **左键单击** | 揭开格子 |
| **右键单击** | 插小旗 / 取消小旗 |
| **左键 + 右键同时按** | 在已揭开的数字格上快速展开周围格子（Chord） |
| **中键单击** | 同上，Chord 展开 |
| **双击** | 在已揭开的数字格上，若周围旗数等于该格数字，展开周围未插旗格 |

### 操作细节

- **左键**：只能揭开未插旗的隐藏格；踩到雷则游戏结束。
- **右键**：只在未揭开的格子上有效；已揭开的格子右键无反应。
- **Chord / 双击**：仅在已揭开的数字格（1~8）上有效。当周围小旗数量等于该格数字时，可：
  - 同时按下左键和右键
  - **双击**该数字格  
  以上操作会自动揭开周围所有未插旗的隐藏格。若其中有雷，则游戏失败。

> 提示：Chord 与双击效果相同，可按习惯任选。在数字格上按住左键后再按右键，或直接双击，均可快速展开。

---

## 5. 游戏规则简述

1. **目标**：揭开所有没有雷的格子，同时避免踩雷。
2. **数字含义**：格子上的数字表示周围 8 格中雷的数量。
3. **首击安全**：第一次点击的格子及其周围 8 格 guaranteed 无雷。
4. **空白扩散**：揭开数字为 0 的格子时，会自动连锁揭开相邻空白区域。
5. **插旗**：右键标记你认为有雷的格子，便于计数和 Chord 操作。
6. **胜利**：所有非雷格都被揭开。
7. **失败**：踩到雷，所有雷会显示出来，触发的雷格会高亮。

## 6. 状态栏说明

| 表情 | 含义 |
|------|------|
| 🙂 | 等待开始（未点击） |
| 😐 | 游戏中 |
| 😎 | 胜利 |
| 💀 | 失败 |

点击表情按钮可随时重新开始。

**雷数计数**示例：

- 10 颗雷、未插旗 → 显示 `010`
- 10 颗雷、插了 3 面旗 → 显示 `007`
- 插旗过多 → 可能显示负数，如 `-002`（与 Windows 扫雷一致）

**计时器**：从第一次左键点击开始，胜利或失败后停止。

## 7. 常见问题

**Q：直接双击 `index.html` 打不开或报错？**  
A：请使用 HTTP 服务器访问（见第 1 节），不要直接用 `file://` 打开。

**Q：右键点击弹出浏览器菜单？**  
A：正常情况下已阻止默认菜单。若仍出现，请确认在棋盘格子上右键，而非页面空白处。

**Q：Chord / 双击没反应？**  
A：请确认：(1) 游戏进行中；(2) 点击的是已揭开的数字格（不是空白或隐藏格）；(3) 周围插旗数等于该格数字；(4) 双键同时按下，或直接双击该格。

**Q：高级难度棋盘太宽？**  
A：可横向滚动查看，或缩小浏览器缩放比例。

**Q：游戏结束后还能点格子吗？**  
A：不能。请点击「新游戏」或表情按钮重新开始。
